附录佐证材料（原创开源软件） 一、功能介绍 原创软件 Traffic - Analysis - Base – DPDK 为基于 DPDK 的高性 能加密流量采集分析系统。 该系统可划分为5 个模块： 系统导航模块、 流量采集模块、业务分类模块、异常检测模块、IP 画像模块。 (1) 系统导航模块主要包含系统引导功能。 (2) 流量采集模块包含流量采集存储功能、流量下载功能、流 量详情查看功能、 流特征提取功能。 高性能流量采集与存储主要是基 于 DPDK 进行高速流量采集，实现对网络数据中心高速流量的捕获 与采集，同时以流（含有相同的源 IP 地址、宿 IP 地址、源端口、宿 端口、 协议的数据包集合） 为单位提取流量的时空特征， 并将提取的 时空特征持久化到流量数据库中，为后续的流量分析提供数据基础。 (3) 加密流量业务分类模块包括业务类别映射功能、文件上传 功能、 特征提取功能、 业务分类模型训练功能、 业务分类功能。加 密 流量业务分类为实现针对加密流量的特定业务分类需求， 提出自动化 在线加密流量采集与样本构建、 业务分类模型构建的两阶段技术实现 系统功能。 自动化在线加密流量采集与样本构建部分将顺序执行自动 化样本采集与初步标记、 样本标签构建、 样本提纯与特征提取。 基于 异构特征的业务分类则分为基于轻量级模型的高效业务分类、 基于深 度学习模型的高精度业务分类和基于多模型联合的均衡业务分类。 (4) 加密流量异常检测模块 包括查看异常检测历史功能、启动 异常检测功能、 查看异常检测详情功能。 加密流量异常检测以高性能 流量采集存储和加密流量业务分类两部分的实现为前提， 在此基础上 实现对加密流量的异常检测。 加密流量异常检测依据真实网络场景下 的设备级流量提出多种异常情况和相应评判策略， 并基于此构建流量 异常检测模型， 并系统实现异常检测功能， 在在线监测中及时对异常

---PAGE 29---
情况进行预警，满足本系统在面向业务分类的异常检测需求。 (5) IP 画像模块包括生成 IP 画像功能、地图显示功能。以 IP 加密流量异常检测提出设备级流量画像的构建， 从系统数据库的构建 层面实现了流量信息及预测结果的时空粒度划分， 实现了本项目对于 业务分类结果时空划分的要求。针对流量分析领域的概念漂移问题， 基于自演进策略设计了设备级流量画像及业务分类模型的生命周期 维持方法。 该系统以先进性、创新性理论应用于项目实施，实现 流量采集 —— 业务分类—— 异常检测全流程的支持服务提供， 及时应对服务设 备的流量行为异常， 保障网络与信息系统运行的有效性和可靠性。 同 时， 以增量式自演进策略实现画像、 模型的动态更新， 保证异常检测 功能的生命周期长久性， 保障真实网络环境进行持续化加密流量监管。 1.1 工作原理 首先， 针对流量标签样本采集存在背景流量干扰问题， 实现网络 流量采集与样本构建方法， 通过自动化提取时间窗口内的进程PID 与 网络端口之间的映射关系， 实现进程与网络流量的对应， 在真实端系 统场景下对各个进程的流量进行隔离性自动化流量采集与初步样本 构建。 其次， 软件实现面向原始采集的加密流量进行特征提取操作。 同 时， 选择性清除弱相关流量数据， 过滤特定加密流量， 保留业务分类 高度敏感特征，实现样本提纯与特征提取。 再次， 将经过样本提纯与特征提取的流量数据特征导入基于异构 特征的多级业务分类功能模块中， 充分利用异构特征和样本标签， 分 别采用传统集成学习、 深度学习等分类模型进行训练， 生成训练样本 数据集。 随后，启动基于 DPDK 的流量采集功能，根据流量采集的实际 需求及 DPDK 框架的特性， 将几种不同的任务分别绑定到不同的CPU
